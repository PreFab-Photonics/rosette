title: DrcResult
---

Result of running DRC.

Returned by `run_drc()`. Contains a list of violations and summary statistics
about the check. Use the `passed` attribute for a quick pass/fail check.

```python
result = run_drc(cell, rules)
if result.passed:
    print(f"DRC passed ({result.polygons_checked} polygons, {result.rules_checked} rules)")
    if result.warning_count:
        print(f"  ({result.warning_count} warnings â€” review recommended)")
else:
    print(f"DRC failed with {result.error_count} errors, {result.warning_count} warnings")
    for v in result.violations:
        print(f"  [{v.severity}] {v.message}")
```

## Attributes

<PyAttribute name={"violations"} type={"list[DrcViolation]"}>

List of all DRC violations found, including both errors and warnings.

</PyAttribute>

<PyAttribute name={"passed"} type={"bool"}>

`True` if no error-severity violations were found.

Warnings (see [`DrcRules.warning_margin`](./DrcRules)) do not cause this to
be `False` â€” a run with only warnings still passes. `rosette drc` and
`rosette check` use this attribute for their exit code.

</PyAttribute>

<PyAttribute name={"error_count"} type={"int"}>

Number of error-severity violations.

</PyAttribute>

<PyAttribute name={"warning_count"} type={"int"}>

Number of warning-severity violations.

</PyAttribute>

<PyAttribute name={"polygons_checked"} type={"int"}>

Number of polygons checked during the DRC run.

</PyAttribute>

<PyAttribute name={"rules_checked"} type={"int"}>

Number of rules evaluated.

</PyAttribute>

<PyAttribute name={"elapsed_ms"} type={"float"}>

Elapsed time in milliseconds.

</PyAttribute>

<PyAttribute name={"suppressed_violations"} type={"int"}>

Number of violations suppressed by `drc_skip` post-filtering.

A violation is suppressed iff every cell it names has `drc_skip = True`
(or is within the subtree of a trusted cell). Violations with unknown
cell-name provenance are always kept (safe default). See
[`Cell.drc_skip`](./Cell).

**Known v1 limitation.** Per-polygon rules (`MinWidth`, `MinArea`, etc.)
and cross-layer pairwise rules (e.g. `Enclosure`, `RequireOverlap`,
cross-layer `MinSpacing`) currently produce violations without cell-name
provenance and are not counted here. This will improve as runner
provenance is filled in.

</PyAttribute>

<PyAttribute name={"skipped_cells"} type={"int"}>

Number of unique cells in the skipped-cell closure for this run â€” that is,
cells with `drc_skip = True` plus every cell reachable from them via
`CellRef`. See [`Cell.drc_skip`](./Cell).

</PyAttribute>
