---
title: Cell
---

A cell containing geometry and references to other cells.

Cells are the primary container for layout geometry. Each cell has a unique
name and holds polygons, paths, text labels, ports, and references to other
cells. Use `cell.at(x, y)` to create positioned instances for ergonomic
placement and port queries.

```python
cell = Cell("my_design")
cell.add_polygon(Polygon.rect(Point.origin(), 10, 5), Layer(1, 0))
cell.add_port(Port("in", Point(0, 2.5), Vector2(-1, 0), width=0.5))

# Position and add to a parent cell
top = Cell("top")
top.add_ref(cell.at(100, 50))
```

## Attributes

<PyAttribute name={"name"} type={"str"}>

Cell name (unique within a design).

</PyAttribute>

<PyAttribute name={"path_length"} type={"float | None"}>

Path length metadata. Automatically set when a cell is created from a Route
via `to_cell()`. Can also be set manually for component cells.

</PyAttribute>

<PyFunction name={"add_path"} type={"(points, width, layer, end_type=None) -> None"}>

Add a path (centerline with width) to the cell.

Paths are an alternative to polygons for representing waveguides and
similar structures. They store a centerline and width, which can be
more compact than storing the full polygon outline.

<Callout title={"Example"} type={"example"}>

```python
cell.add_path(
    [Point(0, 0), Point(100, 0), Point(100, 50)],
    width=0.5,
    layer=1,
    end_type=PathEndType.ROUND
)
```

</Callout>

<div>

<PyParameter name={"points"} type={"list[Point]"}>

List of Point objects along the path centerline.

</PyParameter>
<PyParameter name={"width"} type={"float"}>

Width of the path.

</PyParameter>
<PyParameter name={"layer"} type={"Layer | int | tuple[int, int]"}>

Target layer.

</PyParameter>
<PyParameter name={"end_type"} type={"PathEndType | None"} value={"None"}>

Path end type. Defaults to `PathEndType.FLUSH`.

</PyParameter>

</PyFunction>

### Bounding box

<PyFunction name={"bbox"} type={"() -> BBox | None"}>

Calculate the bounding box of the geometry directly in this cell.

Includes polygons and paths.  Does **not** resolve cell references —
if this cell contains CellRefs or AREFs, their contribution is ignored.
Use [`Library.cell_bbox(name)`](./Library#cell_bbox) for the fully
resolved bounding box of a cell inside a library.

Returns `None` if the cell has no direct geometry.

<PyFunctionReturn type={"BBox | None"} />

</PyFunction>

### Placement

<PyFunction name={"at"} type={"(x, y) -> Instance"}>

Create a positioned instance of this cell.

This is the recommended way to place cells in a design. The returned
Instance tracks the cell reference, allowing port queries without
redundantly passing the cell.

<Callout title={"Example"} type={"example"}>

```python
gc_cell = grating_coupler(layer=layers.silicon.layer)
gc_in = gc_cell.at(0, 0)
gc_out = gc_cell.at(0, 127)

# Get ports directly from instances
port_in = gc_in.port("opt")
port_out = gc_out.port("opt")
```

</Callout>

<div>

<PyParameter name={"x"} type={"float"}>

X coordinate.

</PyParameter>
<PyParameter name={"y"} type={"float"}>

Y coordinate.

</PyParameter>

</div>

<PyFunctionReturn type={"Instance"}>

An Instance positioned at (x, y).

</PyFunctionReturn>

</PyFunction>

<PyFunction name={"add_ref"} type={"(ref) -> None"}>

Add a cell, cell reference, or instance to this cell.

When adding a Cell or Instance, the child cell is automatically
tracked so that `write_gds()` can collect the full hierarchy without
a manual cell list.

<Callout title={"Example"} type={"example"}>

```python
top.add_ref(gc_cell.at(0, 0))        # Instance at position
top.add_ref(route.to_cell("wg"))     # Cell at origin
```

</Callout>

<div>

<PyParameter name={"ref"} type={"Cell | CellRef | Instance"}>

A Cell (placed at origin), Instance, or CellRef to add.

</PyParameter>

</div>

<PyFunctionReturn type={"None"} />

</PyFunction>

### Metadata

<PyFunction name={"add_bend"} type={"(radius, x, y, requested_radius=None) -> None"}>

Add a bend info entry to the cell metadata.

This is used by component authors to record bend locations for
design checks (`run_checks`). The bend radius and location are
stored so that minimum bend radius violations can be detected.

<div>

<PyParameter name={"radius"} type={"float"}>

Effective bend radius in um.

</PyParameter>
<PyParameter name={"x"} type={"float"}>

X coordinate of bend location.

</PyParameter>
<PyParameter name={"y"} type={"float"}>

Y coordinate of bend location.

</PyParameter>
<PyParameter name={"requested_radius"} type={"float | None"} value={"None"}>

Original requested radius, if the router auto-reduced it.

</PyParameter>

</div>

<PyFunctionReturn type={"None"} />

</PyFunction>

<PyAttribute name={"bends"} type={"list[dict]"}>

Bend info entries as list of dicts with keys: `radius`, `x`, `y`,
and optionally `requested_radius`.

</PyAttribute>
