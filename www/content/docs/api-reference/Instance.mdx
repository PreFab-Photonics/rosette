---
title: Instance
---

A cell placed at a specific location with optional transformations.

Instance solves the ergonomic problem of needing to pass the Cell twice:
once to create a CellRef, and again to query ports. With Instance, the
Cell reference is preserved, allowing direct port queries.

```python
# Old pattern (redundant):
gc_ref = CellRef(gc_cell).at(0, 0)
port = gc_ref.port("opt", gc_cell)  # Must pass gc_cell again

# New pattern (ergonomic):
gc_in = gc_cell.at(0, 0)            # Returns Instance
port = gc_in.port("opt")            # No redundancy!
```

Instances can be added directly to cells and support transform chaining:
```python
gc = gc_cell.at(100, 50)
top.add_ref(gc)
```

<Callout type="warn" title="Transform chaining order">

Each chained call wraps the *outside* of the accumulated transform.
The *first* call in the chain is applied first to the geometry.

```python
# .at(10, 0).rotate(90) -> translate first, THEN rotate around origin
# Point (0,0) becomes (10,0) then rotates to (0,10) -- NOT at (10,0)!

# To rotate a component then place it at a specific position,
# rotate first, then translate:
inst = cell.at(0, 0).rotate(90).at(25, 50)  # rotate, then move to (25,50)
```

</Callout>

</PyAttribute>

</PyAttribute>

</PyAttribute>

</PyAttribute>

<PyFunction name={"array"} type={"(columns, rows, col_spacing, row_spacing) -> Instance"}>

Set array repetition (columns × rows rectangular grid with given pitch).

Creates a GDS AREF — a single compact array reference instead of
many individual references. In the viewer, the entire array is
selected and moved as one object.

For hex packings or any skewed / non-orthogonal grid, use
`array_vectors` instead.

<Callout title={"Example"} type={"example"}>

```python
# 10×5 array with 20 µm column pitch and 15 µm row pitch
arr = unit_cell.at(0, 0).array(10, 5, 20.0, 15.0)
top.add_ref(arr)
```

</Callout>

<div>

<PyParameter name={"columns"} type={"int"}>

Number of columns (>= 1).

</PyParameter>
<PyParameter name={"rows"} type={"int"}>

Number of rows (>= 1).

</PyParameter>
<PyParameter name={"col_spacing"} type={"float"}>

Column pitch — center-to-center distance between adjacent copies
along local +X, in µm. Negative values place copies along local −X.

</PyParameter>
<PyParameter name={"row_spacing"} type={"float"}>

Row pitch — center-to-center distance between adjacent copies
along local +Y, in µm. Negative values place copies along local −Y.

</PyParameter>

</div>

<PyFunctionReturn type={"Instance"}>

A new Instance with array repetition set.

</PyFunctionReturn>

</PyFunction>

<PyFunction name={"array_vectors"} type={"(columns, rows, col_vector, row_vector) -> Instance"}>

Set array repetition from arbitrary column and row displacement
vectors.

Lower-level constructor supporting non-orthogonal lattices — hex
packings, skewed test arrays, etc. Vectors are defined in the
instance's local (pre-transform) coordinate space, in µm.

<Callout title={"Example"} type={"example"}>

```python
import math
from rosette import Vector2

# Hex packing (flat-top): adjacent rows staggered by pitch/2.
pitch = 10.0
arr = unit_cell.array_vectors(
    6, 4,
    Vector2(pitch, 0.0),
    Vector2(pitch / 2.0, pitch * math.sqrt(3.0) / 2.0),
)
top.add_ref(arr)
```

</Callout>

<div>

<PyParameter name={"columns"} type={"int"}>

Number of columns (1 to 32767).

</PyParameter>
<PyParameter name={"rows"} type={"int"}>

Number of rows (1 to 32767).

</PyParameter>
<PyParameter name={"col_vector"} type={"Vector2"}>

Column displacement — the offset between copy `(c, r)` and
`(c+1, r)`, in µm.

</PyParameter>
<PyParameter name={"row_vector"} type={"Vector2"}>

Row displacement — the offset between copy `(c, r)` and
`(c, r+1)`, in µm.

</PyParameter>

</div>

<PyFunctionReturn type={"Instance"}>

A new Instance with array repetition set.

</PyFunctionReturn>

</PyFunction>

<PyFunction name={"port"} type={"(name, col=0, row=0) -> Port"}>

Get a transformed port from this instance.

Unlike `CellRef.port()`, this doesn't require passing the Cell again
since the Instance already knows its cell definition. Both position
and direction are fully transformed (translation, rotation, mirroring).

For arrayed instances (see `array` / `array_vectors`), pass `col`
and `row` to address a specific copy in the lattice. The default
`(0, 0)` returns the port of the anchor copy, matching the
behaviour of non-arrayed instances.

<Callout title={"Example"} type={"example"}>

```python
gc = gc_cell.at(100, 50)
opt_port = gc.port("opt")  # Transformed port position & direction

# 180-degree rotation flips both position and direction:
flipped = gc_cell.at(0, 0).rotate(180).at(50, 0)
p = flipped.port("opt")   # direction is now (-1, 0)

# Arrayed: address a specific copy.
bank = ring_cell.array(8, 1, 30.0, 0.0)
p = bank.port("in", col=3)
```

</Callout>

<div>

<PyParameter name={"name"} type={"str"}>

Name of the port to retrieve.

</PyParameter>
<PyParameter name={"col"} type={"int"} value={"0"}>

Grid column of the copy to query (0-indexed). Only meaningful on
arrayed instances.

</PyParameter>
<PyParameter name={"row"} type={"int"} value={"0"}>

Grid row of the copy to query (0-indexed). Only meaningful on
arrayed instances.

</PyParameter>

</div>

<PyFunctionReturn type={"Port"}>

The port with position and direction transformed into world space
for the requested copy.

</PyFunctionReturn>

</PyFunction>

<PyFunction name={"copies"} type={"() -> Iterator[ArrayCopy]"}>

Iterate over the individual copies in this instance's array.

Yields one [`ArrayCopy`](./ArrayCopy) per grid position, in
column-major order (`col` varies fastest). Each yielded object
exposes `col`, `row`, a world-space `transform`, and a `port(name)`
convenience — without mutating this instance or adding any extra
GDS references.

For a non-arrayed instance this yields exactly one copy at
`(col=0, row=0)`, so code written against `copies()` works
uniformly regardless of whether `array()` was called.

Equivalent to `iter(instance)`.

<Callout title={"Example"} type={"example"}>

```python
bank = ring_cell.array(8, 1, 30.0, 0.0)
top.add_ref(bank)                        # one AREF

for copy in bank.copies():
    top.add_text(
        f"R{copy.col}",
        copy.port("in").position,
        layer=Layer(10, 0),
    )
```

</Callout>

<PyFunctionReturn type={"Iterator[ArrayCopy]"}>

Iterator yielding one ArrayCopy per grid position.

</PyFunctionReturn>

</PyFunction>

<PyFunction name={"to_ref"} type={"() -> CellRef"}>

Convert to a CellRef for use with low-level APIs.

The transform is decomposed into GDS-compatible components
(mirror, rotation, translation) following the GDS convention.

<PyFunctionReturn type={"CellRef"}>

A CellRef with the same cell name and transform.

</PyFunctionReturn>

</PyFunction>
