---
title: DrcRules
---

Builder for DRC rule sets.

DrcRules uses a fluent builder pattern — each method returns a new `DrcRules`
with the rule added, so you can chain calls together. Start with an empty
`DrcRules()` and add rules for minimum width, spacing, area, enclosure,
overlap, angles, and more.

```python
rules = (
    DrcRules()
    .min_width(Layer(1), 0.12, name="WG.W.1")
    .min_spacing(Layer(1), Layer(1), 0.15, name="WG.S.1")
    .min_area(Layer(1), 0.01)
    .min_enclosure(Layer(11), Layer(10), 0.10, name="VIA_ENC")
    .allowed_angles(Layer(1), [0, 90], name="WG.ANG")
)

result = run_drc(cell, rules)
```

<PyFunction name={"snap_to_grid"} type={"(layer, grid_pitch, name=None) -> DrcRules"}>

Add a snap-to-grid check for a layer. Verifies all polygon vertex coordinates are
multiples of the manufacturing grid pitch. Off-grid geometry causes mask fracturing
errors and is rejected by most foundries.

Unlike other per-polygon rules, snap-to-grid checks run on fully-transformed (world)
coordinates so that off-grid translations of cell instances are caught.

Common grid pitches: `0.001` (1 nm) or `0.005` (5 nm).

In TOML config, use `snap_to_grid = 0.001` in a per-layer section.

<div>

<PyParameter name={"layer"} type={"Layer | int | tuple[int, int]"}>

Target layer.

</PyParameter>
<PyParameter name={"grid_pitch"} type={"float"}>

Grid pitch in design units (e.g., `0.001` for 1 nm, `0.005` for 5 nm). Must be positive.

</PyParameter>
<PyParameter name={"name"} type={"str | None"} value={"None"}>

Optional rule name for violation reporting.

</PyParameter>

</div>

<PyFunctionReturn type={"DrcRules"}>

A new DrcRules with the rule added.

</PyFunctionReturn>
